Document: Development Challenges and Technical Learnings
=========================================================

This document summarizes the major technical challenges, implementation mistakes, debugging process, and architectural decisions encountered during the development of this project.

---------------------------------------------------------
1. Initial Project Architecture
---------------------------------------------------------

- Initially, the project structure was not well organized.
- Agent definitions, task definitions, and execution logic were developed simultaneously, making debugging difficult.
- Later, the project was restructured into separate modules including:
  - agents
  - tasks
  - database
  - tools
  - utils

---------------------------------------------------------
2. CrewAI LLM Configuration
---------------------------------------------------------

Challenge:

CrewAI attempted to use OpenAI by default.

Error:

OPENAI_API_KEY is required

Root Cause:

No LLM was explicitly configured.

Resolution:

Configured CrewAI to use Ollama instead of OpenAI by creating a centralized config.py file and loading the local Llama3 model.

---------------------------------------------------------
3. Agent Design
---------------------------------------------------------

Initial Issue:

Agents were created without assigning a dedicated LLM instance.

Resolution:

Each agent was updated to use the centralized Ollama configuration.

Each agent now contains:

- Role
- Goal
- Backstory
- verbose=True
- allow_delegation=False

---------------------------------------------------------
4. Task Context Passing
---------------------------------------------------------

Challenge:

Understanding how CrewAI shares information between agents.

Resolution:

Implemented task context passing.

Policy Guardian
↓

Fee Calculator

↓

Document Architect

using

context=[policy_task]

---------------------------------------------------------
5. Local Knowledge Base
---------------------------------------------------------

Challenge:

Assignment required a fallback mechanism.

Resolution:

Created passport_data.json as a local knowledge base containing

- 2026 Passport Fees
- Required Documents
- Eligibility Rules
- Delivery Types
- Passport Page Types

---------------------------------------------------------
6. Web Scraper
---------------------------------------------------------

Challenge:

Assignment required scraping with fallback.

Resolution:

Implemented scraper.py.

Workflow:

Website Available
↓

Use Website

Website Failed
↓

Use Local JSON Database

---------------------------------------------------------
7. Business Logic Separation
---------------------------------------------------------

Initially:

Most logic depended on LLM reasoning.

Improvement:

Moved deterministic logic into Python.

Examples:

- VAT Calculation
- Total Fee
- Passport Validity Validation
- Eligibility Checking
- Markdown Table Generation

---------------------------------------------------------
8. Passport Validation
---------------------------------------------------------

Implemented policy validation.

Rules:

Age < 18
Maximum Validity = 5 Years

18–65
Maximum Validity = 10 Years

Age > 65
Maximum Validity = 5 Years

Invalid requests terminate execution with a clear error message.

---------------------------------------------------------
9. Fee Calculation
---------------------------------------------------------

Fee is no longer generated by the LLM.

Fee is now retrieved directly from the local database using

- Passport Pages
- Passport Validity
- Delivery Type

VAT is calculated separately.

---------------------------------------------------------
10. Document Generation
---------------------------------------------------------

Document checklist is generated dynamically.

Examples:

Adult
Government Employee
Minor Applicant

Duplicate entries are automatically removed.

---------------------------------------------------------
11. Markdown Report Generation
---------------------------------------------------------

Instead of displaying plain text only,

the application generates

output/report.md

which contains

- Passport Readiness Report
- AI Analysis
- Bangla Summary

---------------------------------------------------------
12. Interactive User Input
---------------------------------------------------------

Originally:

All inputs were hardcoded.

Improvement:

Runtime user input was implemented using Python input().

---------------------------------------------------------
